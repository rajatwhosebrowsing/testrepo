RoadEye AI – End-to-End System Workflow

Real-Time Road Damage Detection, Geo-Mapping & Civic Automation

1. Dataset Collection
Gather pothole, crack, manhole datasets from multiple sources covering day/night, multi-weather, multi-view and motion scenarios.
Data Acquisition
2. Dataset Inspection & Format Analysis
Inspect image formats (JPEG, PNG) and annotation formats (JSON, XML, TXT). Identify class labels and inconsistencies.
Data Understanding
3. Dataset Pre-processing
Homogenize datasets into YOLO format. Perform resizing, normalization, class remapping and annotation cleanup.
Preprocessing
4. Data Augmentation
Apply motion blur (Gaussian), low-light simulation, noise, weather effects and perspective changes to improve real-world robustness.
Robust Training
5. Dataset Fusion
Combine regular, night-time, multi-weather and multi-angle datasets into a unified training set with balanced classes.
Data Engineering
6. Model Selection
Choose suitable YOLO version (YOLOv8/YOLOv9/YOLOv10) based on accuracy, latency and mobile deployability.
Model Design
7. Model Training (PC)
Train and fine-tune the YOLO model on a PC/GPU. Optimize hyperparameters for precision, recall and inference speed.
Training
8. Evaluation & Validation
Validate model using test images/videos. Measure precision, recall, mAP, FPS and failure cases.
Validation
9. Model Conversion
Convert trained model to ONNX / TensorFlow Lite for mobile deployment. Apply quantization and pruning if required.
Optimization
10. Deployment Strategy
Choose between web-based inference (server-side) or native Android app (Kotlin) with on-device GPU/NNAPI acceleration.
Deployment
11. Camera & Mounting Setup
Select stable phone mount and camera angle. Optimize shutter speed, frame skipping and vibration handling for bike-mounted usage.
Hardware Integration
12. Real-Time Inference Pipeline
Capture live camera feed, perform inference, smooth detections, filter noise and log damage events in real time.
Edge AI
13. GIS Integration
Attach GPS coordinates using Android Location Services. Assign label IDs, store in database and process spatial data using GeoPandas.
Geo-Mapping
14. Visualization & Analytics
Display detections using Folium maps with markers, heatmaps, severity coloring and route-wise damage tracking.
Visualization
15. Deduplication & Quality Control
Cluster detections spatially and temporally to avoid duplicate reports. Filter false positives and low-confidence detections.
Reliability
16. Autonomous Complaint Agent
Use GenAI & Agentic AI to draft complaints using annotated snapshots, GPS data and severity. Provide UI to send via email, social media or portals.
Automation
17. User Dashboard
Maintain drive history, detections, maps and complaint logs. Enable review, export and audit of collected data.
Application Layer
18. Monitoring & Future Scaling
Monitor performance, retrain with new data, scale to multiple riders, cities and smart-infrastructure platforms.
Scalability
